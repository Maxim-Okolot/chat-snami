# С нами (chat-snami)

Веб-чат и сообщество на движке **[mpchat](https://mpchat.com/)**: общение в реальном времени, форум, профили, магазин, галерея и дополнительные модули. Репозиторий содержит шаблоны, клиентский код и статические ресурсы сайта [snami.mpchat.com](https://snami.mpchat.com/).

## Возможности

| Модуль | Описание |
|--------|----------|
| **Чат** | Общая лента, приват, смайлы, антифлуд, модерация, радио, викторина |
| **Форум** | Темы, сообщения, голосование за посты |
| **Профили** | Анкеты, рейтинг, лайки, просмотры |
| **Магазин** | Покупка ников, цветов, иконок и других предметов за внутреннюю валюту |
| **Галерея** | Загрузка и просмотр фотографий |
| **Кланы** | Страницы кланов и списки участников |
| **Стена любви** | Признания и пожелания |
| **Бот «Снамик»** | Гороскоп и вспомогательные команды в чате |
| **Интеграции** | Webhook голосований, API гороскопа, мост голосов форума |

## Стек

- **Backend:** PHP, mpchat, MySQL, файловое хранилище (`.sys`, `.dat`)
- **Frontend:** JavaScript (ES6+), jQuery, CSS
- **Сборка:** Node.js (утилиты в `scripts/`, без npm-зависимостей в продакшене)

## Требования

- PHP с сессиями и MySQLi
- MySQL
- Установленный движок **mpchat** на сервере (`/usr/share/php/mpchat/`)
- Apache с `mod_rewrite` и `.htaccess` **или** эквивалентная конфигурация Nginx
- Node.js 18+ — только для локальной разработки и утилит

## Развёртывание

1. Клонируйте репозиторий в каталог сайта на сервере с mpchat.
2. Создайте локальный `config.php` (не в git) с параметрами окружения: БД, ключи, пути движка.
3. Убедитесь, что mpchat подключается из `index.php` (`include("/usr/share/php/mpchat/ini.php")`).
4. Настройте права на запись для каталогов данных mpchat (`data/`, аватары, загрузки).
5. Проверьте rewrite: запросы без файла должны уходить в `index.php`; путь `/img/*` — в `assets/img/*`.

Файлы `*.sys` и `config.php` исключены из git (`.gitignore`). Рабочие данные чата живут на сервере и не должны попадать в репозиторий.

## Разработка

### Клиентский JavaScript чата

| Файл | Назначение |
|------|------------|
| `data/jscripts.dat` | **Источник правды** — ядро чата, подставляется в шаблон как `%scripts%` |
| `assets/js/script.js` | Зеркало для локальной отладки; синхронизируется скриптом |
| `assets/js/snamik.js` | Бот «Снамик»; подключается **до** `%scripts%` в `chat.inc` |
| `assets/js/chat-ui.js` | Современный UI: боковые панели, футер, тёмная тема, смайлы |

После изменений ядра чата:

```bash
node scripts/sync-script-js.js
```

Монолитная сборка (legacy, один файл `%scripts%`):

```bash
node scripts/build-jscripts.js
# → data/jscripts.monolith.dat
```

Проверка связки шаблона и скриптов:

```bash
node scripts/verify-chat-ui.js
```

### Стили и тема

- `assets/css/style.css` — базовые стили чата
- `assets/css/site-theme.css` — тёмная тема и оформление standalone-страниц
- Страничные стили: `shop.css`, `forum.css`, `gallery.css`, `clan.css`, `smile-panel.css` и др.
- Standalone-страницы подключают `site-theme-boot.js` и класс `{page}-body` на `<body>`

Пересборка фрагмента тёмной темы из `style.css`:

```bash
node scripts/build-site-theme-css.js
```

### Шаблоны mpchat

Страницы описываются в `data/*.inc`. Параметр `?inc=` выбирает шаблон (например, `?inc=forum`, `?inc=gallery`, `?inc=clan`). Основной чат — `data/chat.inc`.

## API и вспомогательные endpoint'ы

| Endpoint | Назначение |
|----------|------------|
| `?snamik_horoscope=` | Гороскоп для бота (через `api/horoscope.php`) |
| `?forum_api=` | Мост голосований форума |
| `vote-webhook.php` | Webhook внешних голосований |
| `forum_post_votes.php` | Голоса за посты форума |
| `profile_like.php` | Лайки профилей |
| `get_top_photos.php`, `vote_photo.php` | Рейтинг фотографий |

## Безопасность

- `.htaccess` запрещает прямой доступ к `*.inc`, `*.dat`, `*.sys`
- Секреты и локальные override'ы — только в `config.php` и `api/yandex_translate.local.php` (в `.gitignore`)
- Не коммитьте дампы `data/*.sys` с пользовательскими данными

## Лицензия

Код сайта и кастомизации распространяется в рамках проекта «С нами». Движок mpchat — отдельный продукт со своими условиями использования.
